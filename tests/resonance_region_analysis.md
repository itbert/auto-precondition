# Resonance Region Analysis

Короткая инструкция к скрипту [`resonance_region_analysis.py`](auto-preconditioner/tests/resonance_region_analysis.py).

## Что делает скрипт

Скрипт анализирует поведение системы в окрестности резонансной области по частоте:

- строит отклик системы по частоте;
- находит точки усиления (`peak`) и ослабления (`trough`);
- сравнивает число итераций и невязку в этих точках и в соседних частотах;
- показывает, как на это влияют разные предобуславливатели и солверы;
- сохраняет таблицы и графики для дальнейшего анализа.

Идея простая: понять не только где отклик системы усиливается или ослабляется, но и где решение линейной системы становится вычислительно сложнее.

## Быстрый запуск

Из корня репозитория:

```bash
python tests/resonance_region_analysis.py \
  --dataset small \
  --output-dir tests/output_resonance_small
```

Полный запуск на большом датасете со всеми методами, включая `SVD`:

```bash
python tests/resonance_region_analysis.py \
  --dataset large \
  --ignore-size-limits \
  --output-dir tests/output_resonance_large_all
```

## Что лежит в output

В каталоге `output_dir` появляются:

- `summary.txt`
  - краткая текстовая сводка;
  - выбранные точки усиления и ослабления;
  - сравнение этих точек с соседними частотами;
  - какие пары `preconditioner + solver` сильнее всего выделяются по итерациям и невязке.

- `response_scan.csv`
  - отклик системы по всем просмотренным частотам;
  - нужен, чтобы увидеть, где находятся пики и провалы отклика.

- `selected_extrema.csv`
  - список выбранных интересных точек;
  - это частоты, вокруг которых дальше строится локальный анализ.

- `frequency_solve_records.csv`
  - сырые результаты по каждой частоте и для каждой пары `preconditioner × solver`;
  - основной файл для анализа итераций, невязки и времени.

- `neighborhood_summaries.csv`
  - сравнение каждой выбранной точки с её соседями;
  - показывает, насколько в точке итерации или невязка хуже или лучше, чем рядом.

- `response_vs_omega.png`
  - общий график отклика по частоте;
  - на нём отмечены точки усиления и ослабления.

- `iterations_vs_omega__*.png`
  - графики итераций по частоте для каждого солвера;
  - это главный набор графиков, если важна оптимизация.

- `residual_vs_omega__*.png`
  - графики невязки по частоте для каждого солвера;
  - нужны, чтобы понять, не ухудшается ли качество решения в проблемных областях.

- `local_iterations__*.png`, `local_residual__*.png`
  - локальные графики вокруг конкретной интересной точки;
  - помогают понять, это одиночный выброс или широкая область.

- `hist_iterations__*.png`, `hist_residual__*.png`
  - гистограммы значений в окрестности выбранной точки;
  - помогают посмотреть распределение итераций и невязки рядом с пиком или провалом.
